# OpenCode Prompt：Lobuddy UI Redesign

Use the `frontend-visual-polisher` skill.

请严格按照 `Lobuddy_ui-redesign-plan.md` 执行 Lobuddy 桌宠 UI 重设计。

## 项目定位

Lobuddy 不是传统工具软件，而是一个 **companion-first desktop pet assistant**。

优先级：

```text
情绪价值 > 宠物身份 > 陪伴体验 > 功能入口
```

目标是把当前 UI 从“工具型聊天软件 + 宠物按钮”改造成“温暖、可爱、可长期停留在桌面上的 AI 桌宠小伙伴”。

## 当前主要问题

请先基于现有代码和截图理解这些问题：

1. 桌宠本体不像宠物，更像状态监控器
2. 聊天窗口像普通工具软件，陪伴感不足
3. 设置窗口像开发者表单，API 配置过于突出
4. 桌宠、聊天窗口、设置窗口三者视觉风格割裂
5. 用户无法清晰地自定义宠物图标和主题

## 本次硬性要求

### 1. 必须支持用户自定义宠物图标

设置页必须提供：
- 选择本地图片
- 预览图片
- 保存图片路径
- 恢复默认宠物图标
- 启动时读取配置并恢复
- 修改后立即同步到桌宠显示

建议支持：
- `.png`
- `.jpg`
- `.jpeg`
- `.webp`
- `.gif` 如技术成本较高，可先预留字段和结构，优先完成静态图片

注意：
- 不要拉伸变形
- 加载失败必须回退默认图标
- 不要写死路径
- 必须复用项目现有配置系统

### 2. 必须支持用户自定义主题

设置页必须提供：
- 内置主题选择
- 自定义主题色
- 自定义背景色
- 自定义强调色
- 主题预览
- 恢复默认主题
- 保存后重启仍然生效
- 修改后同步到桌宠、聊天面板、设置面板

至少提供主题：
- Cozy Orange
- Sakura Pink
- Mint Green
- Night Companion

注意：
- 不要在各组件里硬编码颜色
- 需要抽出主题 token / style helper
- 所有主要 UI 使用同一套主题系统

## 设计方向

采用：

```text
cozy pet-game warmth
cute desktop companion aesthetic
soft productivity UI
```

参考气质：
- Animal Crossing 的温暖感
- Neko Atsume / Tamagotchi 的宠物陪伴感
- Raycast 的紧凑交互
- Notion 的轻阅读感

避免：
- 企业 SaaS 风格
- 管理后台风格
- 原始开发者工具风格
- 过度渐变
- 过度玻璃拟态
- 随机 AI 味装饰

## 执行步骤

### Step 1：先审计，不要直接改

请先阅读项目结构，找出：
- 桌宠小组件相关文件
- 聊天/主面板相关文件
- 设置窗口相关文件
- 配置读写相关文件
- 当前样式/QSS/主题相关文件

然后输出：
- 当前 UI 文件结构判断
- 最小改动文件列表
- 配置同步方案
- 主题 token 方案
- 自定义宠物图标实现方案

### Step 2：建立主题和资源基础

实现或整理：
- 主题 token
- 主题预设
- 主题应用函数
- 默认宠物资源
- 自定义宠物图标加载函数
- 配置读写同步

### Step 3：先做自定义宠物图标

实现：
- 设置页选择图片
- 图片预览
- 保存路径
- 桌宠即时更新
- 重启恢复
- 恢复默认
- 加载失败兜底

### Step 4：再做自定义主题

实现：
- 主题预设
- 主题色配置
- 主题即时应用
- 三个 UI 表面统一主题
- 重启恢复
- 恢复默认

### Step 5：重设计桌宠小组件

目标：
- 宠物图标成为视觉中心
- 弱化/移除绿色 OK 状态块
- 增加心情气泡
- 使用柔和状态条
- 快捷功能入口宠物化

### Step 6：重设计陪伴面板

目标：
- 顶部展示宠物头像、名字、心情
- 快捷动作使用温暖中文文案
- 聊天气泡柔和
- 记忆/技能卡片化
- 输入区更亲切

快捷动作文案示例：
- 陪我聊聊
- 帮我记住
- 帮我查一下
- 看看你会什么
- 设置小窝

### Step 7：重设计设置小窝

目标：
- 基础设置、外观与装扮、主题设置、陪伴设置、高级设置分组
- 高级设置默认折叠
- API Key / Base URL / Model 不要占据主视觉
- 自定义宠物图标和主题是一级设置

## 约束

严禁：
- 重写 Agent 后端逻辑
- 重写 LLM 调用链路
- 重写记忆系统
- 大规模迁移项目目录
- 引入重型 UI 框架
- 为了 UI 改造破坏现有功能
- 新增一套孤立配置，导致 env/config/settings 不同步

必须：
- 复用现有配置系统
- 保持核心功能不变
- 每个阶段可验证
- 尽量小步改动
- 优先可维护和可继续扩展

## 验证要求

完成后必须验证：

1. 应用能启动
2. 桌宠能显示
3. 聊天窗口能打开
4. 设置窗口能打开
5. 用户可以自定义宠物图标
6. 自定义宠物图标重启后保持
7. 用户可以切换主题
8. 自定义主题重启后保持
9. 三个界面主题一致
10. 聊天核心功能不变
11. 设置保存/取消正常
12. 图片路径失效不会崩溃

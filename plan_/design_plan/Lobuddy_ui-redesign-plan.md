# Lobuddy 专用 UI Redesign Plan

> 目标：把 Lobuddy 从“带宠物壳子的工具型聊天软件”重设计为“陪伴优先、情绪价值优先、功能自然隐藏在互动里的桌面宠物助手”。

---

## 0. 必须先读

本计划用于指导 OpenCode / AI Coding Agent 改造 Lobuddy 的桌宠 UI。

本次不是简单换颜色、加圆角、加阴影，而是一次 **产品形态 + 信息架构 + 视觉系统** 的重设计。

核心原则：

```text
宠物感 > 工具感
陪伴感 > 信息密度
情绪反馈 > 冷冰冰状态
用户自定义 > 固定死样式
轻量可维护 > 过度重构
```

---

## 1. 当前 UI 的主要问题

根据当前截图，Lobuddy 存在以下问题：

### 1.1 桌宠本体不像宠物

当前左侧绿色 `OK` 状态块更像系统监控面板，而不是一个有生命感的桌宠。

问题：
- 缺少宠物形象中心
- 缺少情绪状态
- 缺少陪伴感
- 等级条和按钮像工具面板
- 图标按钮分散，整体不够可爱

### 1.2 聊天窗口太像普通工具软件

当前聊天窗口的橙色顶部栏、消息区域、输入框整体像传统聊天工具，而不是桌宠的小窝。

问题：
- 工具感太强
- 信息密度偏高
- 系统能力说明太直白
- “我的记忆”等内容像文档说明，而不是陪伴型表达
- 缺少宠物头像、心情、当前状态等角色化元素

### 1.3 设置窗口太像开发者表单

当前设置窗口直接暴露 API Key、Base URL、Model、Timeout 等字段，普通用户会感觉这是技术工具，而不是桌宠产品。

问题：
- API 配置占据主视觉
- 缺少基础设置、宠物设置、陪伴设置、主题设置分组
- 没有“自定义宠物图标”和“自定义主题”的清晰入口
- 与桌宠和聊天窗口视觉风格割裂

### 1.4 三个界面风格割裂

目前：
- 桌宠小组件是一套风格
- 聊天窗口是一套风格
- 设置窗口是系统原生表单风格

需要统一为一套温暖、柔和、可爱的桌宠设计语言。

---

## 2. 最终产品定位

Lobuddy 应该被设计成：

> 一个可长期停留在桌面上的陪伴型 AI 桌宠。它首先是一只“陪着用户的小伙伴”，其次才是能聊天、记忆、查资料、写代码、运行命令的 AI 助手。

最终用户感受：

```text
这不是一个聊天工具。
这是一只住在桌面上的小伙伴。
它会陪我、记得我、帮我做事，也能被我装扮。
```

---

## 3. UI 目标状态

本次 UI 应该被重构为三个核心状态：

---

### 3.1 Idle Pet Widget：桌面常驻宠物态

目标：
- 轻量悬浮
- 可爱
- 低打扰
- 一眼看出是“宠物”
- 可被用户自定义形象和主题

必须包含：
- 宠物图标 / 宠物图片 / GIF 展示区域
- 宠物名字
- 心情状态
- 等级 / 亲密度 / 能量等轻量成长信息
- 主要快捷入口
- 展开聊天面板的明确交互

建议交互：
- 单击：选中 / 显示小提示
- 双击：打开陪伴面板
- 右键：打开快捷菜单
- 悬停：显示心情气泡
- 拖拽：移动桌宠位置
- 设置中可选择是否置顶、透明度、大小

---

### 3.2 Companion Panel：展开后的陪伴面板

目标：
- 像打开宠物的小窝，而不是打开一个普通聊天软件
- 有温暖聊天体验
- 功能入口清晰但不压迫

必须包含：
- 顶部宠物身份区：
  - 宠物头像
  - 宠物名字
  - 当前心情
  - 当前状态文案，例如“今天也要一起加油哦”
- 快捷动作区：
  - 陪我聊聊
  - 帮我记住
  - 帮我查一下
  - 看看你会什么
  - 设置小窝
- 聊天区：
  - 左侧宠物消息带头像
  - 右侧用户消息
  - 柔和气泡
  - 清晰时间/状态
- 记忆/技能卡片：
  - 我的记忆
  - 我会的技能
  - 最近对话
  - 今日陪伴状态
- 输入区：
  - 更友好的 placeholder
  - 发送按钮更柔和
  - 支持 Enter 发送，Shift + Enter 换行
  - 保留已有功能逻辑

---

### 3.3 Friendly Settings：友好的设置小窝

目标：
- 像“照顾和装扮宠物”，不是填写开发者配置
- 自定义宠物图标和主题必须是一等功能
- 技术配置收纳进高级设置

设置页面必须重组为以下分组：

#### A. 基础设置
- 宠物名字
- 显示模式
- 是否开机启动
- 是否置顶
- 窗口透明度
- 默认语言

#### B. 外观与装扮
- 自定义宠物图标 / 图片 / GIF
- 选择默认宠物形象
- 图片预览
- 恢复默认形象
- 宠物显示大小
- 宠物动作/动画开关

#### C. 主题设置
- 内置主题选择
- 自定义主题色
- 自定义背景色
- 自定义强调色
- 自定义圆角强度
- 自定义透明度 / 玻璃感开关
- 预览当前主题
- 恢复默认主题

#### D. 陪伴设置
- 主动问候开关
- 互动频率
- 心情反馈开关
- 记忆开关
- 提醒偏好
- 祝福语 / 口头禅

#### E. 高级设置（默认折叠）
- LLM API Key
- LLM Base URL
- LLM Model
- Task Timeout
- Shell Tool 开关
- 调试日志开关

注意：
- 高级设置默认折叠
- API 配置不能占据主视觉
- 普通用户打开设置时，第一眼应该看到宠物名字、形象、主题，而不是 API Key

## 4. 必须支持：用户自定义宠物图标

这是本次改造的硬性要求。

### 4.1 功能要求

用户必须可以在设置页中：
- 选择本地图片作为宠物形象
- 支持常见格式：
  - `.png`
  - `.jpg`
  - `.jpeg`
  - `.webp`
  - `.gif`
- 预览当前宠物形象
- 一键恢复默认宠物形象
- 修改后立即同步到桌宠主界面
- 重启后仍然保持用户选择

### 4.2 GIF 支持

如果当前技术栈是 PySide6：

- 静态图片可使用 `QPixmap`
- GIF 动画优先使用 `QMovie`
- 如果 GIF 支持成本较高，可先实现：
  - PNG/JPG/WEBP 完整支持
  - GIF 作为后续增强项
- 但设置结构必须预留 GIF 支持字段

### 4.3 图片展示规则

自定义宠物图标不能被简单拉伸变形。

需要：
- 等比缩放
- 居中裁切或完整适配
- 圆角/阴影与整体 UI 风格一致
- 对透明 PNG 保持透明
- 图片加载失败时自动回退默认宠物形象

### 4.4 配置字段建议

实际字段名根据项目现有配置系统调整，但必须表达以下语义：

```env
PET_AVATAR_PATH=
PET_AVATAR_TYPE=default/custom
PET_AVATAR_SIZE=normal
PET_AVATAR_ANIMATION_ENABLED=true
```

如果项目已有 `.env` / settings manager / config service：
- 必须复用现有配置读写系统
- 不要新增一套孤立配置
- UI 修改后必须持久化
- 应用启动时必须从配置加载并同步到 UI

## 5. 必须支持：用户自定义主题

这是本次改造的硬性要求。

### 5.1 功能要求

用户必须可以在设置页中：
- 选择内置主题
- 自定义主题色
- 自定义背景色
- 自定义强调色
- 预览主题效果
- 一键恢复默认主题
- 修改后立即应用到：
  - 桌宠小组件
  - 陪伴面板
  - 设置窗口
- 重启后仍然保持主题选择

### 5.2 内置主题建议

至少提供 4 个预设主题：

#### 1. Cozy Orange（默认）
- 温暖奶油白
- 柔和橙色
- 桃粉点缀
- 适合当前 Lobuddy 默认风格

#### 2. Sakura Pink
- 樱花粉
- 浅米白
- 柔和粉色强调
- 偏可爱陪伴

#### 3. Mint Green
- 薄荷绿
- 浅灰白
- 清新绿色强调
- 偏清爽治愈

#### 4. Night Companion
- 暖暗色背景
- 柔和橙/紫点缀
- 适合夜间使用

### 5.3 主题 Token

不要在各组件里到处硬编码颜色。需要抽出统一主题 token。

建议 token：

```python
THEME = {
    "background": "#FFF8EF",
    "surface": "#FFFFFF",
    "surface_soft": "#FFF1DF",
    "primary": "#FF8A3D",
    "primary_soft": "#FFD8B8",
    "text": "#4A2E1F",
    "text_muted": "#8A6F5A",
    "border": "#F1D9C0",
    "shadow": "rgba(124, 76, 32, 0.12)",
    "success": "#8BCF7A",
    "warning": "#F5B84B",
    "danger": "#FF7B7B",
}
```

如果是 Qt/PySide6：
- 可以用统一样式生成函数
- 不要在每个 widget 里重复写完整 QSS
- 推荐建立 `theme.py` / `styles.py` / `design_tokens.py`

### 5.4 配置字段建议

实际字段名根据项目现有配置系统调整，但必须表达以下语义：

```env
THEME_PRESET=cozy_orange
THEME_PRIMARY_COLOR=#FF8A3D
THEME_BACKGROUND_COLOR=#FFF8EF
THEME_SURFACE_COLOR=#FFFFFF
THEME_TEXT_COLOR=#4A2E1F
THEME_RADIUS=large
THEME_TRANSPARENCY=0.96
```

要求：
- 设置页修改主题后立即应用
- 设置页保存后持久化
- 应用重启后读取主题并恢复
- 没有配置时使用默认主题

## 6. 视觉系统规范

### 6.1 默认主题方向

默认使用：

```text
Cozy Orange / 奶油橙 / 暖色治愈系
```

适合桌宠陪伴产品。

建议视觉特征：
- 背景：奶油白 / 暖白
- 主色：柔和橙
- 点缀：桃粉 / 浅黄
- 文本：深棕 / 暖灰
- 边框：浅米色
- 阴影：轻柔暖色阴影

### 6.2 圆角

桌宠产品建议圆角更大，但要统一。

建议：
- 小按钮：12px
- 输入框：14px
- 卡片：18px
- 面板：22px
- 桌宠浮窗：24px+

### 6.3 阴影

避免硬阴影。

建议：
- 使用轻柔扩散阴影
- 不要使用强黑色投影
- 浮窗阴影应表现“轻轻浮在桌面上”

### 6.4 字体与文字

中文界面要可读，不要过度花哨。

建议：
- 标题：稍粗
- 正文：清晰
- 次级信息：暖灰
- 状态文案：有角色感

状态文案示例：
- 工作辛苦啦～
- 我在这里陪你
- 今天也要加油哦
- 要不要和我聊聊？
- 我正在帮你想～
- 记住啦，下次我会提醒你

## 8. 配置同步要求

本项目之前存在“设置页修改不同步到 env / env 不同步到设置页”的问题，因此本次 UI 改造必须顺手规范设置同步逻辑。

要求：
1. 应用启动时从现有配置源加载设置
2. 设置窗口打开时显示真实配置值
3. 用户修改设置后保存到配置源
4. 保存后立即通知相关 UI 刷新
5. 桌宠图标、主题、名字等必须即时生效
6. 重启应用后配置仍然存在

严禁：
- 设置页只改 UI 变量，不持久化
- 写死默认值覆盖用户配置
- 新增一套孤立配置文件导致状态不同步
- 不处理图片路径失效

## 9. 文件组织建议

实际文件名以项目为准。OpenCode 必须先审计项目结构，再决定修改文件。

推荐新增/整理：

```text
ui/
  theme.py              # 主题 token、预设主题、QSS 生成
  assets.py             # 默认宠物资源、图片加载、路径校验
  pet_widget.py         # 桌宠小组件
  companion_panel.py    # 主聊天/陪伴面板
  settings_panel.py     # 设置小窝
  components.py         # 通用按钮、卡片、输入框样式辅助
```

如果项目已有对应文件：
- 优先在现有文件中小步修改
- 不要为了重命名而大规模搬迁
- 不要破坏已有 import

- 不要改 LLM 调用链路
- 不要重写记忆系统
- 不要大规模改项目目录
- 不要引入重型 UI 框架
- 不要做复杂 Live2D

- 不要做账号系统

## 14. 最重要的设计判断

如果某个功能会让界面看起来更像工具，但不是必须显示在主界面，就收纳起来。

如果某个设计能增强“宠物陪伴感”，且不破坏功能，就优先保留。

最终标准：

> 用户打开 Lobuddy 时，第一反应应该是“这只小宠物好可爱”，而不是“这是一个 API 配置工具”。
